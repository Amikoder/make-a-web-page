Unit One.Zero

The Internet

People with computers download browers; browsers, using HTTP, send out queries and return webpages plucked from super-space-saving filing-cabinet computers called servers...hidden in air conditioned closets.

These pages are usually formatted using HTML, hypertext markup language, and can contain almost anything: text, images, videos, links, pdf's, multiple levels. The browser reads the text, markup and references to turn the information into something user-friendly.

Elements

Elements are used in markup, which controls how a webpage, and its content, look. Elements consist, usually, of an opening tag, content, and a closing tag. Some elements, called void tags, don't need closing tags.

Line break: {br}
and the image element: {img src="url" alt="text"}
are two examples of void tags

Bold, and/or Strong: {b}text{/b}, {strong}text{/strong}
need closing tags

Elements can be either inline or block. Inline elements create no additional space around them and can "butt" right up against text. Block elements create a box in which their content resides.

Line break {br} is inline, but paragraph {p}{/p}

is block text

and created the space preceeding and following the above line.

But both are necessary to manage the "white space" on an HTML page, since spaces and carriage returns are rendered null when a browser reads HTML. Multiple spaces and any carriage returns are turned into a single space. Space, therefore, has to be deliberately coded in.

Attributes, span/div
Attributes modify their elements, or are properties of elements. Span and Div, for instance, can have attributes added to them. Both elements contain text, Span is inline and Div is block. Neither are void tags. Div behaves much like the paragraph tag, but doesn't create as much space around it.

Unit One.Two

Stealing Code from Other Pages

Looking at the code of existing web pages provides excellent examples of code-in-action. Not only can you see the end product, but how the elements and style are formatted correctly to follow HTML (or CSS) syntax and rules.

Google Chrome's developer tools provides user-friendly ways to sift through complex DOMs, as well as see how coders are using CSS to minimize redundancy. At first glance, all that code hidden behind little triangles make it seem easy. And then you click that nesting triangle and fall down the rabbit hole. Deeper and deeper. Oh, and they make it easy to pick out boxes. Boxes, boxes, everywhere.

Programmer's text editors also make learning/performing programming easier. Programs for writing programs! Or web pages, in this case. They will, for instance, provide a corresponding closing tag for your opening one.

Unit One.Three

CSS

As mentioned previously, CSS controls how an HTML page looks. It also saves coders substantial time by allowing them to apply a repetitive style to multiple elements of the same type.

There are three ways to add style to a page:
1. Writing it in the head element of the HTML page.
2. Within the head element, add a link to a seperate CSS file. This keeps large projects organized, and looks something like this:

{link: rel="stylesheet" type="text/css" href="main.css"}

3. Writing the style inline with the HTML. Not the best choice, leads to a lot of repetition.

Some Particulars of CSS

CSS both helps avoid repetition and maintain consistency by applying a singular style to an unlimited number of instances of the same element. This, handily enough, also makes implementing changes easier. It's also ruled by an abstract concept called inheritance. In a page of CSS code, the most specific rule applies. If a class has a specific attribute, and there's a class nested within it without any attributes specified, the nested class shares the attribute of the parent (ancestor) class.

A CSS Element:
h1 { background-color: black; }

"h1" is the selector.
"background-color: black;" is the declaration
"background-color is the property
and "black" is the value
